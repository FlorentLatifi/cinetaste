# Accessibility notes (SPA)

CineTaste aims for usable keyboard and screen-reader flows on the core product surfaces.

## Implemented

| Area | Behavior |
|------|----------|
| Skip link | ‚ÄúSkip to main content‚Äù on authenticated shell |
| Focus | `:focus-visible` rings on buttons, links, inputs |
| Reduced motion | Honors `prefers-reduced-motion` |
| High contrast | `data-contrast` theme; follows `prefers-contrast: more` unless user overrides; toggle in shell + Account + auth cards; stored as `ct_contrast` |
| Search | Labeled field, live status, results heading receives focus after search, single accessible name per result card |
| For You | List semantics, action buttons named with title, reasons labeled |
| Title detail | Action group labels, similar titles as a list |
| Onboarding | Progressbar, live progress, action labels include title, rate panel focuses first rating option |
| Errors | `role="alert"` on form/API failures |

## Automated gate (CI)

Guest routes are scanned with **axe-core** via Playwright after `npm run build`:

```bash
cd frontend
npm ci
npm run build
npx playwright install chromium   # once locally
npm run test:a11y
```
