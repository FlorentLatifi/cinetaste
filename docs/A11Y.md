# Accessibility notes (SPA)

CineTaste aims for usable keyboard and screen-reader flows on the core product surfaces.

## Implemented

| Area | Behavior |
|------|----------|
| Skip link | “Skip to main content” on authenticated shell |
| Focus | `:focus-visible` rings on buttons, links, inputs |
| Reduced motion | Honors `prefers-reduced-motion` |
| High contrast | `data-contrast` theme; follows `prefers-contrast: more` unless user overrides; toggle in shell + Account + auth cards; stored as `ct_contrast` |
| Search | Labeled field, live status, results heading receives focus after search, single accessible name per result card |
| For You | List semantics, action buttons named with title, reasons labeled |
| Title detail | Action group labels, similar titles as a list |
| Onboarding | Progressbar, live progress, action labels include title, rate panel focuses first rating option |
| Errors | `role="alert"` on form/API failures |

## Automated gate (CI)

Guest routes are scanned with **axe-core** via Playwright after `npm run build`:

```bash
cd frontend
npm ci
npm run build
npx playwright install chromium   # once locally
npm run test:a11y
```

| Route | Ready marker |
|-------|----------------|
| `/login` | “Welcome back” |
| `/register` | “Create account” |
| `/forgot-password` | “Forgot password” |
| `/reset-password` | “Reset password” |
| `/` (For You) | Mocked session + slate |
| `/search` | Search form + results after query |
| `/watchlist` | Watchlist heading |
| `/account` | “Your profile” |
| `/titles/:id` | Title detail |
| `/onboarding` | Mocked incomplete session + cards |

**Authenticated routes** use Playwright `page.route` mocks (`e2e/helpers/mockApi.ts`) — no Postgres/API required.

Tags: `wcag2a`, `wcag2aa`, `wcag21a`, `wcag21aa`.

## Manual checks (quick)

1. Tab through top nav — order: brand → For You → Search → Watchlist → Account → High contrast → Sign out.  
2. Search: type query, Enter, focus moves to “Results” heading; Tab into first title link.  
3. Onboarding: keyboard-only complete Haven’t seen / Rate it / Good.  
4. Toggle OS “reduce motion” — card animations should not play.  
5. Toggle **High contrast** — solid black surfaces, stronger borders, brighter muted text.
